### <a name="accessibility-improvements-in-windows-workflow-foundation-wf-workflow-designer"></a>Windows Workflow Foundation (WF) 工作流程設計工具中的協助工具增強功能

|   |   |
|---|---|
|詳細資料|Windows Workflow Foundation (WF) 工作流程設計工具提升與協助工具技術的運作方式。 這些改良包括下列變更：<ul><li>定位順序變更為從左到右，由上至下在一些控制項：</li><li>設定相互關聯資料的初始化相互關聯視窗<xref:System.ServiceModel.Activities.InitializeCorrelation>活動</li><li>內容定義視窗<xref:System.ServiceModel.Activities.Receive>， <xref:System.ServiceModel.Activities.Send>， <xref:System.ServiceModel.Activities.SendReply>，和<xref:System.ServiceModel.Activities.ReceiveReply>活動</li><li>多個函式是透過鍵盤提供：</li><li>當編輯活動的屬性，屬性可以摺疊群組透過鍵盤它們將焦點放在第一次。</li><li>警告圖示現在是由鍵盤存取。</li><li>現在可由鍵盤存取更多屬性中的按鈕 [屬性] 視窗。</li><li>鍵盤使用者現在可以存取引數和變數 窗格中的工作流程設計工具的標頭項目。</li><li>改良的可見性的焦點，例如當項目：</li><li>在工作流程設計工具與活動設計工具所使用的資料格來加入資料列。</li><li>中的欄位中定位<xref:System.ServiceModel.Activities.ReceiveReply>和<xref:System.ServiceModel.Activities.SendReply>活動。</li><li>設定變數或引數的預設值</li><li>螢幕助讀程式現在可以正確辨識：</li><li>工作流程設計工具中設定中斷點。</li><li><xref:System.Activities.Statements.FlowSwitch%601>， <xref:System.Activities.Statements.FlowDecision>，和<xref:System.ServiceModel.Activities.CorrelationScope>活動。</li><li>內容<xref:System.ServiceModel.Activities.Receive>活動。</li><li>目標類型為<xref:System.Activities.Statements.InvokeMethod>活動。</li><li>例外狀況下拉式方塊和最後一節中<xref:System.Activities.Statements.TryCatch>活動。</li><li>訊息類型下拉式方塊中，加入相互關聯初始設定式 視窗、 內容定義 視窗中和在訊息活動中的 CorrelatesOn 定義 視窗中用來分隔 (<xref:System.ServiceModel.Activities.Receive>， <xref:System.ServiceModel.Activities.Send>， <xref:System.ServiceModel.Activities.SendReply>，和<xref:System.ServiceModel.Activities.ReceiveReply>)。</li><li>狀態機器轉換，並轉換的目的地。</li><li>附註和接頭<xref:System.Activities.Statements.FlowDecision>活動。</li><li>設定活動的內容 （以滑鼠右鍵按一下） 功能表。</li><li>屬性值編輯器、 清除搜尋 按鈕，依類別和按字母順序排序按鈕和運算式編輯器 對話方塊，在屬性方格中。</li><li>在工作流程設計工具中顯示比例百分比。</li><li>中的分隔符號<xref:System.Activities.Statements.Parallel>和<xref:System.Activities.Statements.Pick>活動。</li><li><xref:System.Activities.Statements.InvokeDelegate>活動。</li><li>字典的活動選取型別視窗 (<code>Microsoft.Activities.AddToDictionary&lt;TKey,TValue&gt;</code>，<code>Microsoft.Activities.RemoveFromDictionary&lt;TKey,TValue&gt;</code>等。)。</li><li>[瀏覽並選取.NET 型別] 視窗中。</li><li>在工作流程設計工具中的階層連結。</li><li>選擇高對比佈景主題會看到的工作流程設計工具在可見性中的許多加強功能和其控制項所要的使用者更好對比更值得注意的焦點項目使用的選取方塊項目之間的比例。</li></ul>|
|建議|如果您使用重新裝載工作流程設計工具的應用程式，您的應用程式可以受益於這些變更藉由執行上述其中一個動作：<ul><li>重新編譯應用程式以目標為.NET Framework 4.7.1。 依預設會啟用這些存取範圍變更。</li><li>如果您的應用程式 4.7 或更早版本的.NET Framework 為目標但在.NET Framework 4.7.1 上正在執行，您可以新增以下選擇這些舊版的協助工具的行為不[AppContext 參數](~/docs/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element.md)至<code>&lt;runtime&gt;</code>區段app.config 的檔案，並將它設定為<code>false</code>，如下列範例所示。</li></ul><pre><code>&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;&#13;&#10;&lt;configuration&gt;&#13;&#10;&lt;startup&gt;&#13;&#10;&lt;supportedRuntime version=&quot;v4.0&quot; sku=&quot;.NETFramework,Version=v4.7&quot;/&gt;&#13;&#10;&lt;/startup&gt;&#13;&#10;&lt;runtime&gt;&#13;&#10;&lt;!-- AppContextSwitchOverrides value attribute is in the form of &#39;key1=true/false;key2=true/false  --&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;&lt;/configuration&gt;&#13;&#10;</code></pre>.NET Framework 4.7.1 為目標的應用程式或更新版本並想要保留舊版協助工具的行為也可以選擇使用舊版的協助工具功能的明確設為這個 AppContext 參數<code>true</code>。|
